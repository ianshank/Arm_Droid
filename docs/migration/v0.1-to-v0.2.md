# Migration Guide: armdroid v0.1.x → v0.2.x

This guide covers all breaking changes and recommended import path updates for code written against
armdroid v0.1.x. All old paths remain importable through v0.3.x and emit `DeprecationWarning`.
They will be **deleted in v0.4.0**.

---

## 1. Domain types (state, errors, protocols)

The monolithic `armdroid.protocols` module has been split into three domain modules.

### State types

```python
# v0.1.x (deprecated)
from armdroid.protocols import ArmState, PlanStep, SymbolicState, DetectedObject

# v0.2.x (canonical)
from armdroid.domain.state import ArmState, PlanStep, SymbolicState, DetectedObject
```

### Error types

```python
# v0.1.x (deprecated)
from armdroid.protocols import ArmDriverError, ArmCommandRejected

# v0.2.x (canonical)
from armdroid.domain.errors import ArmDriverError, ArmCommandRejected
```

All error types extend `armdroid.domain.errors.ArmError`. You can now catch all armdroid errors
with a single clause:

```python
from armdroid.domain.errors import ArmError

try:
    await driver.send_joint_positions(angles, duration=1.0)
except ArmError as exc:
    logger.error("arm error", exc_info=exc)
```

### Protocol types

```python
# v0.1.x (deprecated)
from armdroid.protocols import (
    ArmDriverProtocol,
    ArmControllerProtocol,
    ArmPlannerProtocol,
    ArmEnvironmentProtocol,
    ArmPerceptionProtocol,
)

# v0.2.x (canonical)
from armdroid.domain.protocols import (
    ArmDriverProtocol,
    ArmControllerProtocol,
    ArmPlannerProtocol,
    ArmEnvironmentProtocol,
    ArmPerceptionProtocol,
)
```

## 2. Factory functions

```python
# v0.1.x (deprecated)
from armdroid.factory import build_arm_driver, build_arm_controller, build_arm_orchestrator

# v0.2.x (canonical)
from armdroid.orchestration.factory import (
    build_arm_driver,
    build_arm_controller,
    build_arm_orchestrator,
)
```

---

## 3. ArmOrchestrator

```python
# v0.1.x (deprecated)
from armdroid.orchestrator import ArmOrchestrator

# v0.2.x (canonical)
from armdroid.orchestration.orchestrator import ArmOrchestrator
```

---

## 4. ESP32 driver

```python
# v0.1.x (deprecated)
from armdroid.hardware.esp32_json_driver import Esp32JsonDriver

# v0.2.x (canonical)
from armdroid.hardware.esp32 import Esp32JsonDriver
```

Private constants that were previously accessible on the shim module:

```python
# v0.1.x (deprecated)
from armdroid.hardware import esp32_json_driver
esp32_json_driver._KEEPALIVE_POLL_FLOOR_S   # float

# v0.2.x (canonical)
from armdroid.hardware.esp32.driver import _KEEPALIVE_POLL_FLOOR_S, _FIRST_STATE_POLL_INTERVAL_S
```

## 5. Config schema

```python
# v0.1.x (all in one module)
from armdroid.config.schema import (
    ArmSettings,
    ArmConfig,
    TransportConfig,
    MotionConfig,
    ControlConfig,
    PerceptionConfig,
    PlanningConfig,
    SimConfig,
)

# v0.2.x (same paths still work — schema is now a package, imports unchanged)
from armdroid.config.schema import ArmSettings  # still valid
```

No migration needed for `config.schema` — the package `__init__.py` re-exports everything.

## 6. Legacy driver method names (pre-v0.1 API)

If you have code using the old `start/stop/send_joint_command/get_joint_states` method names, use
`LegacyArmDriverAdapter` as a temporary bridge:

```python
from armdroid.compat import LegacyArmDriverAdapter
from armdroid.orchestration.factory import build_arm_driver
from armdroid.config.schema import ArmSettings

cfg = ArmSettings()
modern_driver = build_arm_driver(cfg)
adapter = LegacyArmDriverAdapter(modern_driver)  # emits DeprecationWarning

# Old code continues to work:
await adapter.start()
await adapter.send_joint_command([0.1, 0.2, 0.3, 0.0, 0.0, 0.0])
states = await adapter.get_joint_states()   # returns numpy array
await adapter.stop()
```

Method mapping:

| Legacy (v0.1) | Modern (v0.2) |
|---|---|
| `start()` | `connect()` |
| `stop()` | `disconnect()` |
| `send_joint_command(angles)` | `send_joint_positions(tuple(angles), 2.0)` |
| `get_joint_states()` | `read_state().joint_positions` (as ndarray) |
| `open_gripper()` | `open_gripper()` (unchanged) |
| `close_gripper()` | `close_gripper()` (unchanged) |
| `home()` | `home()` (unchanged) |
| `emergency_stop()` | `emergency_stop()` (unchanged) |
| `clear_emergency_stop()` | `clear_emergency_stop()` (unchanged) |

`LegacyArmDriverAdapter` will be removed in v0.4.0. Migrate to the modern API before then.

## 7. Quick sed/grep migration script

Run this from the repo root to bulk-rename imports in your own codebase:

```bash
# Unix/macOS
find . -name "*.py" -exec sed -i \
  -e 's/from armdroid\.protocols import/from armdroid.domain.state import/g' \
  -e 's/from armdroid\.factory import/from armdroid.orchestration.factory import/g' \
  -e 's/from armdroid\.orchestrator import/from armdroid.orchestration.orchestrator import/g' \
  -e 's/from armdroid\.hardware\.esp32_json_driver import/from armdroid.hardware.esp32 import/g' \
  {} \;
```

> **Note**: `armdroid.protocols` exported state types, error types, *and* protocol types in one
> namespace. After the sed pass, review any imports of error types and redirect them to
> `armdroid.domain.errors` rather than `armdroid.domain.state`.

## 8. Version compatibility matrix

| armdroid version | Old paths work? | Warning level | Adapter available? |
|---|---|---|---|
| v0.1.x | ✅ (native) | — | ❌ |
| v0.2.x (current) | ✅ (shim) | `DeprecationWarning` | ✅ |
| v0.3.x (planned) | ✅ (shim) | `FutureWarning` | ⚠️ compat-only package |
| v0.4.0 (planned) | ❌ **deleted** | — | ❌ |
